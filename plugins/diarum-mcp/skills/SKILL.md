# Diarum MCP 服务器技能

## 概述

Diarum MCP 服务器使 AI 助手能够通过模型上下文协议 (MCP) 与 Diarum 日记应用进行交互。此集成允许 LobeHub 用户使用自然语言命令管理他们的日记条目。

## 功能特性

- **读取日记**：按日期查询日记条目、搜索内容或浏览最近的条目
- **创建/更新日记**：编写新的日记条目或更新现有条目，支持内容、心情、天气和标签
- **AI 驱动的分析**：从日记条目生成定期摘要和洞察
- **文本润色**：通过 AI 辅助编辑增强日记文本
- **标签管理**：使用标签组织和筛选日记
- **统计功能**：追踪写作习惯和连续记录

## 配置

### 环境变量

| 变量 | 描述 | 默认值 |
|------|------|--------|
| `DIARUM_BASE_URL` | Diarum 实例的基础 URL | `http://localhost:8090` |
| `DIARUM_API_TOKEN` | API 认证令牌 | （必填） |

### 认证

1. 启动你的 Diarum 实例
2. 创建账户或使用现有凭据
3. 通过 Diarum 网页界面生成 API 令牌
4. 设置 `DIARUM_API_TOKEN` 环境变量

## 可用工具

### 日记操作

#### `get_diary_by_date`
获取特定日期的日记条目。

**参数：**
- `date`（字符串，必填）：YYYY-MM-DD 格式的日期

**示例：**
```json
{
  "date": "2024-01-15"
}
```

#### `create_or_update_diary`
创建新的日记条目或更新现有条目。

**参数：**
- `date`（字符串，必填）：YYYY-MM-DD 格式的日期
- `content`（字符串，必填）：Markdown 格式的日记内容
- `mood`（字符串，可选）：心情表情符号或文本（如 "开心"、"😊"）
- `weather`（字符串，可选）：天气描述（如 "晴天"、"雨天"）
- `tags`（字符串数组，可选）：用于分类的标签

**示例：**
```json
{
  "date": "2024-01-15",
  "content": "## 今日亮点\n\n- 完成项目里程碑\n- 举行了很棒的团队会议",
  "mood": "高效",
  "weather": "晴天",
  "tags": ["工作", "项目"]
}
```

#### `search_diaries`
按内容搜索日记条目。

**参数：**
- `query`（字符串，必填）：搜索查询
- `limit`（数字，可选）：最大结果数（默认：50）

**示例：**
```json
{
  "query": "团队会议",
  "limit": 10
}
```

#### `get_recent_diaries`
获取最近的日记条目。

**参数：**
- `limit`（数字，可选）：要获取的条目数（默认：5，最大：100）

**示例：**
```json
{
  "limit": 10
}
```

#### `delete_diary`
删除日记条目。

**参数：**
- `id`（字符串，必填）：日记条目 ID

**示例：**
```json
{
  "id": "abc123"
}
```

#### `get_diary_stats`
获取日记统计数据，包括总条目数和当前连续记录。

**参数：**
- `timezone`（字符串，可选）：用于连续记录计算的时区（如 "Asia/Shanghai"）

**示例：**
```json
{
  "timezone": "Asia/Shanghai"
}
```

### 标签操作

#### `get_all_tags`
获取所有标签及其使用次数。

**参数：** 无

#### `get_diaries_by_tag`
获取具有特定标签的所有日记条目。

**参数：**
- `tag`（字符串，必填）：标签名称

**示例：**
```json
{
  "tag": "工作"
}
```

### AI 操作

#### `analyze_diary_period`
生成日记条目在一段时间内的 AI 分析。

**参数：**
- `start_date`（字符串，必填）：开始日期（YYYY-MM-DD）
- `end_date`（字符串，必填）：结束日期（YYYY-MM-DD）
- `keywords`（字符串，可选）：按关键词筛选
- `period_type`（字符串，可选）："week" 或 "month"（默认：自定义范围）

**示例：**
```json
{
  "start_date": "2024-01-01",
  "end_date": "2024-01-31",
  "keywords": "项目",
  "period_type": "month"
}
```

#### `polish_text`
通过 AI 辅助增强日记文本。

**参数：**
- `content`（字符串，必填）：要润色的文本
- `mode`（字符串，可选）："medium"（轻度编辑）、"strong"（深度改写）或 "custom"
- `custom_prompt`（字符串，可选）："custom" 模式的自定义指令

**示例：**
```json
{
  "content": "今天很好。和团队开会。讨论了项目。",
  "mode": "medium"
}
```

### 特殊查询

#### `on_this_day`
获取历史同日期的日记条目。

**参数：**
- `date`（字符串，可选）：要查询的日期（默认：今天）

**示例：**
```json
{
  "date": "01-15"
}
```

#### `random_diary`
获取随机日记条目用于回忆。

**参数：**
- `exclude_date`（字符串，可选）：要排除的日期

**示例：**
```json
{
  "exclude_date": "2024-01-15"
}
```

## 使用示例

### 自然语言命令

用户可以在 LobeHub 中通过自然语言与 Diarum 交互：

1. **读取日记：**
   - "我昨天写了什么？"
   - "显示我上周一的日记"
   - "1月15日我在想什么？"

2. **写日记：**
   - "为今天的锻炼创建一条日记"
   - "用会议记录更新昨天的日记"
   - "给今天的日记添加'健身'标签"

3. **搜索：**
   - "查找所有关于项目截止日期的日记"
   - "搜索提到感到压力的条目"
   - "显示标记为'家庭'的日记"

4. **分析：**
   - "分析我这个月的日记条目"
   - "我的周记中有什么规律？"
   - "生成我一月份条目的摘要"

5. **回忆：**
   - "一年前的今天我在做什么？"
   - "给我看一条过去的随机日记"
   - "我已经连续写了多少天？"

## 错误处理

MCP 服务器处理常见的错误场景：

- **认证错误**：无效或缺少 API 令牌
- **未找到**：日记条目不存在或访问被拒绝
- **验证错误**：日期格式无效或缺少必填字段
- **服务器错误**：Diarum 实例不可用

## 安全注意事项

- API 令牌应安全存储，切勿提交到版本控制
- 在生产环境中使用 HTTPS
- MCP 服务器遵守 Diarum 的访问控制（用户只能访问自己的日记）
- 内容存储在本地的 Diarum 实例中

## 故障排除

### 连接问题
1. 验证你的 Diarum 实例正在运行
2. 检查 `DIARUM_BASE_URL` 是否指向正确的地址
3. 确保 API 令牌有效且未过期

### 认证失败
1. 重新登录 Diarum 刷新会话
2. 如需要，生成新的 API 令牌
3. 验证令牌具有所需权限

### 数据缺失
1. 检查你查询的日期格式是否正确（YYYY-MM-DD）
2. 验证所请求日期的日记条目存在
3. 确保你有权访问所请求的条目
